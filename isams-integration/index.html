Integration Guide

iSAMS Integration — Scope & Coverage

Everything your school needs to know about how Admissions+ and Data+ connect with iSAMS, what data is supported, and how it flows.

How the integration works

Admissions+ connects to your iSAMS system through the iSAMS REST API using secure OAuth 2.0 authentication. Your school's API credentials (Client ID and Client Secret) are configured once in Admissions+, and data transfers are initiated by staff from within Admissions+ when needed.

SECTION 5

Custom fields (User Defined Fields)

Admissions+ fully supports iSAMS custom fields. Any User Defined Field you've configured in iSAMS can be mapped to an Admissions+ form field, and data flows in both directions.

What's supported

  • Read from iSAMS — Custom field values are imported alongside student records, so existing data appears pre-populated in forms
  • Write to iSAMS — When parents or applicants fill in custom fields on your forms, the values are written back to the matching UDF in iSAMS
  • Applicants and students — UDFs work for both iSAMS applicant records and student records
  • Mapping — Fields are matched using the iSAMS custom field identifier, so the right data goes to the right place

How to set up: Map your iSAMS custom fields in the Admissions+ field mapping screen. Each iSAMS UDF is identified by its field ID — select the matching Admissions+ form field, and the integration handles the rest.

SECTION 10

Setup requirements

What you need from iSAMS

  • API Client ID — your iSAMS API application identifier
  • API Client Secret — your iSAMS API application secret
  • School ID — your iSAMS school identifier (the subdomain part of your iSAMS URL, e.g. yourschool from yourschool.isams.cloud)
  • Active API subscription — your iSAMS account must have an active API Services Manager subscription

Hosting variants supported

yourschool.isams.cloud yourschool.edgen.cloud yourschool.isamshosting.cloud Custom endpoint URL

Sync schedule

  • Lookup data (languages, nationalities, etc.) syncs daily
  • Student and contact data imports and exports are run on demand by staff from within Admissions+
  • All transfers are logged and any errors are reported to your school admin
