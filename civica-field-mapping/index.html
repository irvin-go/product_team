Admissions+ to Civica CCM: Supported Fields & Mapping

This document lists the fields the Admissions+ Civica Integration App can write back to Civica, where each one maps, and which fields are mandatory, lookup-controlled, or available when updating an existing record. You choose which fields to send on each import.

How it works
  • One direction: Admissions+ writes to Civica CCM. Civica remains your source of truth.
  • The app creates new records and updates existing ones. It never deletes or blanks data already in Civica.
  • Lookup fields (marked Lookup = Yes) are mapped to your Civica values during the migration step.
Minimum fields to create a record: Forename, Surname, Date of Birth, Gender, Title, Address, Email.

Lookup fields (mapped during migration)

These fields use code lists. During migration you map your current values to the matching Civica values, so imports are accepted:

Gender  |  Title  |  Nationality  |  Religion  |  Ethnicity  |  Disability  |  Country  |  Country of Birth  |  SEN Need  |  Relationship  |  First Language  |  Home Language

Create vs Update

  • Create needs the minimum fields above; everything else is optional.
  • On update, all fields are available except Email, which is read-only after the record is first created.
  • The five Basic Details fields (Forename, Surname, Date of Birth, Gender, Title) are optional on update.
