UNIT-e Integration — Scope & Coverage

Everything your college needs to know about how Admissions+ connects with UNIT-e, what data is supported, and how it flows.

How the integration works

Admissions+ connects to your UNIT-e system through the Applicaa UNIT-e Integration App, a lightweight Windows application installed on your college network. The app securely bridges Admissions+ and your UNIT-e system, transferring applicant, contact, and course data using the UNIT-e WebAPI.

Applicant matching

Before importing, each applicant is automatically sorted into one of two groups so that no duplicate records are created:

  • New applicants — no existing UNIT-e record; a brand-new student and application is created in UNIT-e
  • Existing applicants — already matched to a UNIT-e record; the existing student is updated and a new application is linked

Safe by design: For student and contact data, Admissions+ only ever creates new records or updates existing values in UNIT-e — it does not delete a student's personal or contact records. Lookup values that UNIT-e doesn't recognise are skipped with a warning rather than causing the import to fail.

SECTION 7

Courses and status mapping

Admissions+ courses are mapped to UNIT-e modules before import. The system auto-matches UNIT-e course reference codes to your Admissions+ Provider Course IDs, and saved mappings are restored automatically so you don't need to re-map on every import.

Course status mapping

Each Admissions+ course status is exported as the matching UNIT-e status code:

Admissions+ statusUNIT-e codeUNIT-e status
Interested (Application Form Subjects)OOriginal Application
Offer (Add to Offer List)IInterview Application
Enrolled (Confirmed Subjects)EEnrolment Application

How course sync behaves

  • Duplicate prevention — courses already present in UNIT-e with the same module and status are skipped
  • Keeps courses aligned — course entries removed in Admissions+ are cleared from UNIT-e before new courses are imported, so both systems stay in step
  • Smart warning — if you try to import courses for applicants who haven't been imported yet, the app warns you and offers to import them first
SECTION 8

Lookup data (reference lists)

A dedicated Sync Lookups screen imports your UNIT-e reference lists into Admissions+, so that application form dropdowns match valid UNIT-e values exactly. Only active options are included.

SECTION 10

Setup requirements

What you need from UNIT-e

  • WebAPI access — your UNIT-e WebAPI endpoint URL (e.g. uniteapi.yourcollege.ac.uk)
  • API credentials — a WebAPI login that returns a session key for the integration

What you need from Admissions+

  • Admissions+ login — email, password, and your college subdomain
  • Integration App installer — downloaded from the Admissions+ admin panel

IT requirements

  • Windows desktop — the Integration App runs on Windows
  • Network access — reach both your Admissions+ site and your UNIT-e WebAPI host

How syncs work

  • Data transfers are initiated manually from the Integration App by staff
  • Expired UNIT-e sessions are detected and refreshed automatically, preventing mid-import failures
  • The app auto-updates on each launch and shows a changelog when a new version is available
  • Every import produces a per-student report showing successes, warnings, and errors — viewable in-app or downloadable
